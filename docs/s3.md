# Object Storage (MinIO + S3)

Media uploads (avatars, future user-generated content) go through django-storages with an S3-compatible backend. Locally that's [MinIO](https://min.io/) running in Docker; in production it can be the same MinIO image, AWS S3, or any other S3-compatible service.

There are **two backends** in `apps/base/storage.py`:

| Backend                                 | When to use                                                                                                                |
|-----------------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| `apps.base.storage.S3MediaStorage`      | Default for local dev (MinIO) and recommended for prod. Handles the Docker-internal vs. browser-endpoint URL split.        |
| `apps.base.storage.MediaS3Storage`      | Legacy AWS-S3 path with `ManifestFilesMixin` for hash-named static files. Kept for backwards compatibility with the AWS setup described below. |

## Local Dev (MinIO, the default)

Nothing to configure — `compose.yml` runs MinIO and `just create_env` writes sensible defaults to `.env`:

```bash
DEFAULT_FILE_STORAGE=apps.base.storage.S3MediaStorage
MEDIA_S3_ACCESS_KEY=minioadmin
MEDIA_S3_SECRET_KEY=minioadmin
MEDIA_S3_ENDPOINT_URL=http://minio:9000
MEDIA_S3_URL_ENDPOINT_URL=http://localhost:9000
MEDIA_S3_BUCKET_NAME=media
```

The two endpoint variables are deliberately different:

- `MEDIA_S3_ENDPOINT_URL` — what Django uses to *talk* to MinIO. Inside the Docker network, MinIO is reachable at `http://minio:9000`.
- `MEDIA_S3_URL_ENDPOINT_URL` — what ends up in URLs the browser will fetch. The browser doesn't know about the Docker network, so URLs need to point at `http://localhost:9000`.

`S3MediaStorage` rewrites the host substring at URL-generation time so generated avatar URLs work in the browser.

The `web` container runs `./manage.py ensure_s3_bucket` on startup, which creates the bucket if it doesn't exist.

The MinIO console is at http://localhost:9001 (creds default to `minioadmin` / `minioadmin`).

## Prod with MinIO (or any S3-compatible service)

Same `S3MediaStorage` backend. Just point the env vars at the prod endpoint:

```bash
DEFAULT_FILE_STORAGE=apps.base.storage.S3MediaStorage
MEDIA_S3_ACCESS_KEY=<access key>
MEDIA_S3_SECRET_KEY=<secret key>
MEDIA_S3_ENDPOINT_URL=https://s3.your-provider.com
# Set MEDIA_S3_URL_ENDPOINT_URL only if you need URL rewriting; usually omit in prod.
MEDIA_S3_BUCKET_NAME=<bucket name>
```

Static files (Vite-hashed JS/CSS) are served by WhiteNoise with `Cache-Control: max-age=31536000, immutable` — no S3 hop required.

## Prod with AWS S3 (legacy `MediaS3Storage`)

If you prefer the old AWS-specific path with manifest-hashed static files:

### 1. Create an IAM user and bucket

1. In the [AWS Console](https://console.aws.amazon.com/), create an IAM user with programmatic access. Save the access key and secret.
2. Create an S3 bucket. If you'll point a CNAME at it, use a fully qualified domain name as the bucket name.
3. Attach a policy like the following to the user (replace the bucket name):

   ```json
   {
       "Version": "2012-10-17",
       "Statement": [
           {
               "Effect": "Allow",
               "Action": "s3:ListBucket",
               "Resource": "arn:aws:s3:::your-bucket-name"
           },
           {
               "Effect": "Allow",
               "Action": [
                   "s3:PutObject",
                   "s3:GetObject",
                   "s3:PutBucketCORS",
                   "s3:DeleteObject",
                   "s3:PutObjectAcl"
               ],
               "Resource": "arn:aws:s3:::your-bucket-name/*"
           }
       ]
   }
   ```

4. Add a CORS configuration to the bucket:

   ```json
   [
     {
       "AllowedHeaders": ["Authorization"],
       "AllowedMethods": ["GET", "HEAD"],
       "AllowedOrigins": ["*"],
       "MaxAgeSeconds": 3000
     }
   ]
   ```

### 2. Settings

Add to `.env`:

```bash
DEFAULT_FILE_STORAGE=apps.base.storage.MediaS3Storage
STATICFILES_STORAGE=apps.base.storage.StaticS3Storage
AWS_ACCESS_KEY_ID=<key>
AWS_SECRET_ACCESS_KEY=<secret>
AWS_STORAGE_BUCKET_NAME=<bucket>
AWS_S3_REGION=us-east-2
```

After every static-file change, run `./manage.py collectstatic` to upload to S3.
